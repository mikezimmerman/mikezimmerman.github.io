For a major global financial enterprise environment in which all monitors were calibrated to 1024 x 768, my client needed new-hire training on complex proprietary software systems that already consumed the full screen real estate. We demo'ed the typical off-the-shelf screen-capturing software (Captivate, Articulate, et al) but felt that those solutions shattered the natural feel of the real sytems (making the training feel more like watching an animated Flash Movie, not interacting with the actual software) while imposing an unremovable interface around these system screens that required them to be shrunk and/or cropped—both of which were dealbreakers in regard to the mission-critical need for this training to look and feel like the real deal. So, I created this "floating" instructional UI in JavaScript. It could be dragged anywhere within the screen limits—or "popped" to one side or the other via a single click—so that the entirety of the system screen being instructed remained viewable and able to be interacted with.
The original targeted IE7 (the sole browser used across this entire global enterprise), and was an object-oriented pure JavaScript massive "all in one" solution (as dictated by the business use cases) which made use of my own version of JS "polymorphism" to enable every possible type of instructional UI that we could ever need to be implemented through a single simple interface with an easy-to-program API (also of my creation). I have re-created a very simplified version here in current HTML5/CSS3/JS/jQuery technologies. To view the code behind this project on GitHub, click here.
